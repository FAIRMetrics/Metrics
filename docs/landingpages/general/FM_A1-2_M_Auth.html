FAIR Metric - A1.2 - Metadata - retrieval protocol supports auth

If the resolution protocol supports authentication and authorization for access to restricted content.



A1.2 requires that the protocol used to retrieve research objects clearly describes any necessary authentication and authorisation procedures. This ensures that access conditions are transparent and that restricted content is accessed in a defined and controlled way. Optionally, if a FAIRsharing record for the hosting database is provided, the evaluation first checks the declared [data access conditions](https://fairsharing.gitbook.io/fairsharing/additional-information/database-conditions#data-access-condition). If the access condition is “open”, tests will pass immediately. Next, if access is restricted, the resolution protocol is then evaluated to determine whether it supports authentication and authorisation. If the test does make use of the FAIRsharing record, this can ensure a pass for completely open access databases, even if a test implementation does not recognise a particular protocol. A resource satisfies this metric if access is declared as open in FAIRsharing record metadata, or if restricted access to metadata is supported by appropriate authentication and authorisation mechanisms. Note that identification of the FAIRsharing database record URL/DOI (so that FAIRsharing metadata may be retrieved) can be done in a variety of ways, e.g. using the FAIRsharing API to find possible matches between the identifier and registered databases, hard coding the FAIRsharing record identifier, or expecting it to be provided as a second argument to the test.
